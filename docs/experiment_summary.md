## 5. 主要発見

### 5.1 ELOが最重要特徴量
- Optuna最適化でELO重み=25.6% (最大)
- ELO単体 (elo_only) が現行モデルを accuracy +1.9pp, F1 +6.5pp で上回る
- K=25.3, home_bonus=85.4 は文献と整合

### 5.2 recent_form の予測力は低い
- 現行重み 25.6% → 最適化後 **5.5%** (83%減)
- 380試合で検証するとフォーム効果はノイズに近い
- **注意**: これは「フォームが無意味」ではなく、他の特徴量(ELO等)にフォーム情報が含まれるため

### 5.3 draw予測の構造的欠陥
- **全モデルでdraw予測0件** (elo_onlyを除く)
- 原因: シグモイド変換でdrawが「残差」扱い
- elo_onlyだけがdraw=40件予測 (14件正解) → F1 macro最良

### 5.4 capital_powerの安定性
- 静的指標だが最適化前後で重み変化小 (0.170→0.164)
- 資本力は長期的チーム力の安定した代理変数

## 7. 次の自己改善ループ

### 最優先: draw予測の構造改善
1. シグモイド変換を3クラスsoftmax変換に置換
2. ELO差 → 直接3クラス確率への変換関数設計
3. draw確率がargmaxで選ばれうる構造への移行

### 次点: ELO本番統合
1. EloSystemクラスをpredict_logic.pyに追加
2. Geminiプロンプトに ELO レーティング情報を追加
3. advantage_to_probs の代替として ELO ベース確率変換

### 中期: モデル構造の見直し
1. 線形結合 + シグモイドからの脱却
2. 序数回帰またはmultinomial logistic regression
3. Dixon-Coles ポアソンモデルの検討
