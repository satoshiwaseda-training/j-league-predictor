## 2. 致命的欠陥: draw予測の欠落

### 現象
- 全7モデル中、drawを1件でも予測できたのは **elo_only のみ** (40件)
- 現行モデル (current v5) は **draw 0件**
- Optuna最適化後も **draw 0件**

### 原因分析
1. **base_draw < base_home/base_away**: ベース確率で既にdraw < home, draw < away
2. **シグモイド変換**: home/awayの確率が変動してもdrawは残差(1-h-a)扱い
3. **draw_closeness_boost**: 最適化で0.179に設定されたが、closeness*0.179≈0.08で
   base_draw=0.16+0.08=0.24 < base_home=0.41 → argmaxでhomeに負ける
4. **elo_only成功の理由**: ELO差が小さいとき d = 0.20+closeness*0.15 ≈ 0.30 で
   h, a が小さくなりdrawがargmaxで勝てる

### draw比率の実態
| 年度 | 実際のdraw率 |
|------|-------------|
| 2024 | 27.4% (104/380) |
| 2025 | 25.5% (97/380) |
| 2026 | 22.7% (10/44) |

**約4試合に1試合はdraw** — これを予測できないモデルは構造的に不完全。

## 3. 現行ロジックの真の弱点

### 3.1 draw予測の構造的不可能
上述の通り。シグモイド変換の設計がdrawを「残差」として扱うため、
drawが argmax で選ばれることがほぼ不可能。

### 3.2 ELOの不在
現行モデルにELOがなく、勝点ベースのteam_strengthのみ。
ELO単体でも現行モデルを上回る(accuracy +1.9pp, F1 +6.5pp)。

### 3.3 recent_formの過大評価
現行重み: recent_form=0.256 (最大)
最適化結果: recent_form=0.055 (最小級)
→ 380試合で検証するとフォームの予測力は低い。

### 3.4 capital_powerの安定性
現行重み: 0.170、最適化後: 0.164 → ほぼ維持
静的指標だが安定した予測力を持つ(資本力≈長期的チーム力)。

---

## 4. elo_only が最良ベースラインである理由

1. **動的更新**: 試合ごとにレーティングが変わるため、フォームを内包
2. **home_bonus内蔵**: ELO計算時にhome_bonus=50を加算
3. **draw予測可能**: ELO差が小さい試合でdrawが選ばれる設計
4. **シーズン跨ぎ**: 前シーズンのELOが引き継がれる

## 6. 次のアクション

### 安全に反映可能
1. **EloSystemクラスの追加** (predict_logic.pyへ) — 本番挙動変更なし
2. **バックテスト基盤・ドキュメント** — 評価インフラ

1. **重みの変更** — draw問題が未解決のまま重みだけ変えても本質改善にならない
2. **base確率の変更** — draw構造問題の解決が先

### 次の自己改善ループで回すべき
1. **draw予測モデルの根本改善**: シグモイド変換ではなく、3クラス直接予測への移行
   (例: ELO差 → 3クラスsoftmax変換)
2. **ELOの本番統合**: predict_with_geminiのプロンプトにELOスコアを追加
3. **recent_form重みの見直し**: 0.256→0.05の大幅減が妥当かの検証
