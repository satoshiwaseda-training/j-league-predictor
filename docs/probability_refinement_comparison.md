## 各改善施策の効果

### 1. Dead parameter 0化
- 影響: **数値的にゼロ** (死にパラメータは元から寄与0)
- 意義: コード衛生管理、将来の混乱防止

### 2. ELOモジュールバグ修正 (predict_integrated内のELO受け渡し)
- 旧: ELOスコアがcalculate_parameter_contributionsに渡されていなかった
- 新: elo.score_pair()の結果を正しく渡す
- 影響: **これが最大の改善要因** (log loss -0.014)

### 3. ELO重み 0.116 → 0.13
- val=2025でのスイープ結果: 0.13がF1最良地点
- log loss/Brierも0.12付近で改善が最大

### 4. Temperature scaling
- 不採用 (temp=0.8でlogL微改善だがF1が低下)
- draw#が減少するトレードオフ
