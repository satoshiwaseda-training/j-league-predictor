## 3. 各パラメータの設計根拠

### 3.1 チーム強度 (30%)
- 勝点・順位差に基づくスコア化 (0.0〜1.0)
- ホームは+5pt補正（平均的ホームアドバンテージ相当）
- 最重要指標のため最高重み

### 3.2 直近フォーム (25%)
- 直近5試合: W=1.0, D=0.5, L=0.0
- 加重平均（直近ほど重み大）: `[0.35, 0.25, 0.20, 0.12, 0.08]`
- モメンタム効果を表現

### 3.3 ホームアドバンテージ (15%)
- 固定スコア: ホーム=0.62, アウェー=0.38
- 根拠: Jリーグ統計でホーム勝率≈48%, アウェー勝率≈30%

### 3.4 H2H対戦成績 (10%)
- `(wins + draws×0.5) / total` でスコア化
- 心理的優位性・戦術的相性を反映

### 3.5 選手状態 (10%)
- 怪我/出場停止1人あたり -0.10 ペナルティ
- 最大ペナルティ: -0.50 (5人以上で上限)

### 3.6 天気/疲労 (5%)
- `fatigue_factor`に基づく疲労係数
- 悪天候はホームチーム有利（慣れている）: base=0.55

### 3.7 移動距離 (5%)
- Haversine公式による精密計算
- 疲労段階:
  - 〜300km: 0.00（影響なし）
  - 300〜600km: 0.10（軽微）
  - 600〜1000km: 0.25（中程度）
  - 1000〜1500km: 0.50（大）
  - 1500km超: 0.75（極大）

## 4. 確率変換式

```python
# シグモイド変換
shift = tanh(raw_advantage × 3) × 0.30  # -0.30〜+0.30

# ベース確率（ホームアドバンテージ込み）
base_home = 0.40
base_draw = 0.25
base_away = 0.35
```

raw_advantage = 0 のとき → ホーム40%, 引分25%, アウェー35%
