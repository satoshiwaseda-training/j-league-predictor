## 3. 確率品質分析

### 3.1 log loss の分解
| モデル | log loss | 解説 |
|--------|----------|------|
| elo_only | 1.039 | 確率出力が最も正確 |
| current v5 | 1.057 | drawを25%固定出力→drawの情報損失は少ない |
| integrated+ELO | 1.066 | draw予測を出すことの確率コスト |
| random (1/3) | 1.099 | ベースライン |

### 3.2 log loss微悪化の原因
integrated+ELO の log loss がcurrent v5より高い理由:
1. **draw予測のコスト**: draw=59件を出すが、precision=30.5%で70%は誤り
2. **draw確率の不正確さ**: drawと予測した試合で実際drawは30.5%だが、
   drawと予測しなかった試合でも実際draw率は25%程度 → 判別精度が不十分
3. **closenessの精度**: 死にパラメータがraw_advに混入し、closenessが不正確

### 3.3 改善が困難な理由
- 3ロジットパラメータのgrid searchで改善余地なし
- 根本的な改善には: (a) 死にパラメータの除去、(b) ELO重みの増加、
  (c) closenessの定義改善 が必要 → 全て次ループのスコープ
