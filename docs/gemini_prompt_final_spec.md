# Geminiプロンプト統合最終仕様 (gemini_prompt_final_spec.md)

> 作成日: 2026-04-09
> 対象: predict_logic.py: predict_with_gemini()

---

## 1. 渡す情報

### A. 統計モデル事前確率 (必須)
```
## 統計モデル事前確率 (v7 3ロジット方式)
22パラメータの重み付き分析 + ELOレーティングに基づく事前予測:
  ホーム勝利: {h_pct}%
  引き分け:   {d_pct}%
  アウェイ勝利: {a_pct}%
実力接近度: {closeness:.2f} (1.0=完全均衡, 0.0=一方的)
```

### B. ELO要約 (あれば)
```
ELOレーティング: {home_team} {home_elo:.0f} / {away_team} {away_elo:.0f} (差: {diff:+.0f})
```

### C. draw注意シグナル (closeness >= 0.5 の場合のみ)
```
※ 統計モデルは引き分けの可能性を示唆しています (接近度{closeness:.2f})。引き分け確率を慎重に検討してください。
```

---

## 2. 渡さない情報

- _3LOGIT_PARAMS の内部パラメータ (scale_ha, bias_draw等)
- 個別ロジット値 (解釈困難)
- MODEL_WEIGHTS の生の重み配分
- バックテスト結果 (Geminiを誘導しすぎる)
- temperature scaling の内部詳細

---

## 3. 差し込み位置

既存プロンプト内の以下の位置:
```
## 重み付きパラメータ貢献度 ...
加重合計ホームアドバンテージスコア: ...
移動距離: ...

【ここに差し込み】← NEW

## 直近フォーム ...
```

理由: パラメータ貢献度の集約結果として自然な位置。
フォーム・H2H・天気の前に事前確率を提示することで、
Geminiがこれらの追加情報で事前確率を補正する流れになる。

---

## 4. 指示文への追加

既存の「## 指示」セクションに以下を追加:
```
上記の統計モデル事前確率は22パラメータの重み付き分析に基づく参考値です。
この事前確率を出発点として、直近フォーム・H2H・天気・怪我等の定性情報で
補正した最終予測を出力してください。
統計モデルが引き分けを示唆している場合は、その可能性を軽視しないでください。
```

---

## 5. 過度な誘導の防止

- 「参考値」「出発点」という表現で、Geminiの自律判断を尊重
- 統計モデルの確率を「正解」として押し付けない
- ただしdraw注意シグナルだけは明示的に（drawを無視する傾向への対策）
