# データソース優先順位 (data_source_priority.md)

> 作成日: 2026-04-09

---

## ソース一覧と優先順位

| 優先度 | データ | ソース | 公式 | 備考 |
|--------|--------|--------|------|------|
| 1 | 試合日程・結果 | jleague.jp/match/section | Yes | 唯一のソース |
| 1 | 順位表 | jleague.jp/standings | Yes | 唯一のソース |
| 1 | 規律統計 | jleague.jp/stats | Yes | カード数 |
| 2 | xG統計 | fbref.com | No | J1のみ。J2/J3は中立値 |
| 2 | H2H対戦成績 | シード乱数 | No | 将来: 実データ化 |
| 2 | 怪我情報 | web検索/キャッシュ | No | 精度不安定 |
| - | ELO | 試合結果から自動構築 | 派生 | get_past_resultsに依存 |
| - | 天気 | Open-Meteo | No | 無料API、安定 |

## 採用ルール

1. **同一指標に複数ソースがある場合**: 公式(is_official=True)を優先
2. **公式取得失敗時**: キャッシュ → 補助ソース → 中立値(fallback) の順
3. **ELO**: 試合結果が取得できれば必ず構築。結果0件なら全チーム初期値1500
4. **xG**: J1のみ。取得失敗またはJ2/J3は中立値(0.5, 0.5)
