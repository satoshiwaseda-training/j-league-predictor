> 作成日: 2026-04-09

## 1. 評価方法の制約

Gemini APIは非決定論的(temperature=0.25)かつ有料のため、
大規模バックテストは実施不可能。以下の方法で評価:

- **既存予測ログ分析**: prediction_store.jsonの65件から構造分析
- **v7 refinedバックテスト**: 統計モデル部分の定量評価(val=2025, n=377)
- **構造的検証**: プロンプトに渡される情報の正確性確認

---

## 2. 既存ログ分析

| 項目 | Gemini (2.0 Flash) | 統計モデル |
|------|-------------------|-----------|
| 予測件数 | 28 | 37 |
| 実績付き | 0 | 19 |
| draw予測率 | 10.7% (3/28) | 0.0% (0/37) |
| draw_prob平均 | 28.7% | 25.0% (固定) |
| 正答率 | 計測不能 | 57.9% (11/19) |

### 重要な知見
- **Geminiはdraw_probを平均28.7%で出力**している → J1実績(25.7%)に近い
- しかしargmaxではdraw選択率は10.7%に留まる → home/awayが上回るケースが多い
- 統計モデル(旧v5)はdrawを0件予測 → v7で解消済み

## 3. v7 refined事前確率のGeminiへの効果 (構造的分析)

### 3.1 渡される情報の構造
```
統計モデル事前確率 (v7 3ロジット方式)
  ホーム勝利: {h_pct}%
  引き分け:   {d_pct}%
  アウェイ勝利: {a_pct}%
実力接近度: {closeness:.2f}
ELO期待勝率: {home} {elo_h}% / {away} {elo_a}%
※ draw注意シグナル (接近試合時)
```

### 3.2 期待される効果
1. **draw注意シグナル**: closeness>=0.5かつd_pct>=25%で発動
   - val=2025で推定150-200試合(40-53%)で発動
   - Geminiのdraw過小評価を修正する可能性
2. **事前確率のアンカリング**: Geminiの出発点が統計モデルに近づく
   - 極端な予測(home 80%等)の抑制効果
3. **ELO情報**: 順位/勝点だけでなく動的戦力評価を提供

### 3.3 リスク
- Geminiが事前確率をそのまま返す(思考放棄)可能性
  → 「参考値」「出発点として補正」表現で軽減
